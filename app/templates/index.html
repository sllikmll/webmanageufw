Best practice: для постоянной работы лучше использовать отдельного automation-user или root по ключу. Парольный доступ тоже ок, но только когда реально нужен.

Перед сохранением проверь:
  • порт SSH реально слушает нужный host;
  • тип auth совпадает с тем, как сервер пускает внутрь;
  • если сервер не root-only, то есть sudo password.
